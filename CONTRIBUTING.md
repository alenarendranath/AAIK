# Contributing to AAIK

Thank you for your interest in contributing! All contributions are welcome.

## Guidelines

1. **Original content only** — do not copy content from books, courses, or other copyrighted materials.
2. **Use only public domain or open-source images** in YAML manifests.
3. **No vendor-specific content** — keep examples cloud-agnostic where possible.
4. **Clear explanations** — write for someone learning Kubernetes for the first time.

## How to Contribute

1. Fork the repository
2. Create a branch: `git checkout -b feature/my-topic`
3. Make your changes
4. Submit a Pull Request with a clear description

## Content Standards

- YAML files must be valid and tested
- Each topic folder must have a `README.md` and `commands.md`
- Manifests should include comments explaining each key field
- No references to specific books, paid courses, or proprietary platforms
