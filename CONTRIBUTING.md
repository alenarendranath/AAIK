# Contributing to AleK8s

Thank you for your interest in contributing to **AleK8s**!

## Guidelines

### Original Content Only
All contributions must be **original work**. Do not copy or paraphrase content from:
- Books, courses, or paid learning materials
- Official documentation (unless quoting minimally with attribution)
- Other repositories without explicit permission

### What to Contribute
- New YAML manifest examples covering Kubernetes concepts
- kubectl command references
- Troubleshooting scenarios with diagnosis steps
- End-to-end examples combining multiple concepts
- Corrections to existing content

### YAML Manifest Standards
- Use `nginx:alpine` or `busybox` as demo container images
- Add clear comments explaining every non-obvious field
- Include apply/verify/cleanup commands in the header comment
- Follow the naming convention: `alek8s-<descriptor>`

## Pull Request Process

1. Fork the repository
2. Create a branch: `git checkout -b feature/topic-name`
3. Add your content following the folder structure
4. Ensure your content is original
5. Submit a pull request with a clear description

## Code of Conduct

Be respectful, constructive, and welcoming to contributors of all experience levels.
